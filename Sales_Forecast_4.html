Weekly Sales Forecast for Big Box, Inc.¶

Big Box, Inc. requested a forecast of weekly sales for the individual deparments in each of its stores. The company provided the data team with three CSV files containing data related to its stores and their departments, weekly sales, and economic and weather conditions in the geographic areas where stores are located. Specifically Big Box requested a forecast of the next three months beyond the actual sales data provided.

We performed a time series analysis through the following phases:

Time Series Analysis Phases¶

  1. Data Import, Validation, and Merge
  2. Exploratory Data Analysis: Stores, Departments, and Total Sales
  3. Exploratory Data Analysis: Weekly Sales
  4. Feature Creation and Selection
  5. Modeling and Prediction: Random Forest Regression
  6. Modeling and Prediction: SARIMA

Phase 4. Feature Selection and Creation¶

Correlations Between Features¶

To explore the relationship between features, we created a heatmap showing a correlation matrix. There did not appear to be a strong relationship between the features, with the exception of one pair of Markdown features. There was not a strong correlation between any of the features and weekly sales, which is our target for forecasting.

We noted these relationships:

  • Most features had a low degree of correlation with other features (less than ± 0.30).
  • Looking at the vertical column for weekly sales, which was being forecasted, most other features had near zero correlation. Weekly sales had low correlation with the size of the store (0.24).
  • There were two pairs of features with higher correlations values, namely between the markdowns and whether the week was associated with holiday. For example the holiday weeks and markdown 3 had a moderate correlation (0.50). Furthermore the holiday weeks and markdown 2 had a moderate correlation (0.39).
  • Markdown 1 and Markdown 4 had the only strong correlation (0.84).

Preprocessing, Selection, and Creation of Features¶

We defined and called a function to perform the preprocessing steps needed to prepare the data for modeling. The function performs these steps:

  • Recoded the Date feature as two new numerical features representing the week number and the year. The first date in the provided data set is week number 5 of the year 2010.
  • Recoded the IsHoliday features as numerical 0=False and 1=True.
  • Recoded the store Type as three new features using One Hot Encoding. The new features were numerical TypeA, TypeB, and TypeC whose values are 0=False and 1=True.
  • Dropped the original Type and IsHoliday features, but kept the Date feature temporarily. The Date feature was needed later when splitting the training and validation sets.
  • Dropped the markdown features. This data is very incomplete. Future analysis can include these features when more data is available.

def preprocess(thedataset):

    #Convert the date to a week number
    #The earliest date of 2010-02-05 is in week number 5 of the year
    tempdate = pd.to_datetime(thedataset['Date'])
    thedataset['Year'] = tempdate.dt.year
    thedataset['WeekNum'] = tempdate.dt.isocalendar().week

    #Recode the IsHoliday as False=0 and True=1
    thedataset['IsHolidayInt'] = thedataset['IsHoliday'].astype(int)
    
    #Recode the store type as separate features using one hot encoding
    onehotencoder = OneHotEncoder(handle_unknown='ignore')
    typearray = onehotencoder.fit_transform(thedataset[['Type']]).toarray()
    typedata = pd.DataFrame(typearray)
    thedataset['TypeA'] = typedata[0]
    thedataset['TypeB'] = typedata[1]
    thedataset['TypeC'] = typedata[2]    
    
    #Drop the IsHoliday column because we recoded it as numeric
    thedataset = thedataset.drop(columns=['IsHoliday'])

    #Drop the Type column because we recoded it as numeric
    thedataset = thedataset.drop(columns=['Type'])

    #Drop the markdowns as they are very incomplete.
    #Future analysis can encorporate these when more data is available.
    thedataset = thedataset.drop(columns=['MarkDown1'])
    thedataset = thedataset.drop(columns=['MarkDown2'])
    thedataset = thedataset.drop(columns=['MarkDown3'])
    thedataset = thedataset.drop(columns=['MarkDown4'])
    thedataset = thedataset.drop(columns=['MarkDown5'])

    return thedataset

Create the training set and validation set¶

We split the full feature set into a training set and a validation set based on dates. We dropped the Date column prior to saving the data sets as separate CSV files.

  • The training set contains 80% of the rows and it ranges from 2/5/2010 to 1/31/2012.
  • The validation set contains 13% of the row and it ranges from 4/1/2012 to 7/31/2012.

NOTE: The cutoff dates leave an unused set of 7% of the rows that range from 2/1/2012 to 3/31/2012.
