Weekly Sales Forecast for Big Box, Inc.¶

Big Box, Inc. requested a forecast of weekly sales for the individual deparments in each of its stores. The company provided the data team with three CSV files containing data related to its stores and their departments, weekly sales, and economic and weather conditions in the geographic areas where stores are located. Specifically Big Box requested a forecast of the next three months beyond the actual sales data provided.

We performed a time series analysis through the following phases:

Time Series Analysis Phases¶

  1. Data Import, Validation, and Merge
  2. Exploratory Data Analysis: Stores, Departments, and Total Sales
  3. Exploratory Data Analysis: Weekly Sales
  4. Feature Creation and Selection
  5. Modeling and Prediction: Random Forest Regression
  6. Modeling and Prediction: SARIMA

Phase 6. Modeling and Prediction: SARIMA¶

Modeling was performed using statsmodels' method for seasonal autoregressive integrated moving average (SARIMA). This method is appropriate for time series data that has a seasonal component. Some departments in the company's stores have seasonal sales. Of the provided data, the SARIMA model only used dates and weekly sales. It does not use any other features such as store size and type, unemployment, temperature. etc. This model predicts one department's sales at a time.

Decomposition of a Seasonal Data Set¶

Below are the details of the analysis for Store 1 Department 3 which has a seasonal component. We graphed this department's weekly sales and noted high peaks in late August of each year. We also decomposed the sales data and noted what might be a very small upward trend in sales. However we clearly see a strong seasonal component.

Finding the Best SARIMA Model¶

SARIMA models are denoted by three parameters known as p, d, and q. To find the best model to predict sales for Store 1 Department 3, we fit one model for every combination of the (p, d, q) triplet and then we selected the model with the lowest AIC. The Akaike information criterion (AIC) is a measure of model fit and a lower AIC is better. This was the model chosen:

  • Order = (1, 0, 0)
  • Seasonal_Order = (1, 0, 0, 52)
  • AIC = 92.7255

# Generate AIC scores for the p,d,q combinations
warnings.filterwarnings("ignore") # specify to ignore warning messages
list_AIC = pd.DataFrame({}, columns=['param','param_seasonal','AIC'])

for param in pdq:
    for param_seasonal in seasonal_pdq:
        try:
            mod = sm.tsa.statespace.SARIMAX(onedept_set,
                                            #trend='c',
                                            order=param,
                                            seasonal_order=param_seasonal)
            results = mod.fit()
            
            #Save the AICs so we can easily find the minimum
            temp = pd.DataFrame([[param, param_seasonal, results.aic ]],
                                columns=['param','param_seasonal','AIC'])            
            list_AIC = list_AIC.append(temp, ignore_index=True)
            
        except Exception as e:
            sys.stdout.write('Exception: ' + str(e))
            sys.stdout.flush()            
            continue

Model Fitting and Evaluation¶

We split our data set into a training set, which contained 80% of the data, and a validation set which contained the remaining 20%. The training set contained sales through the end of January 2012, while the validation set contained sales beyond January 2012. We fit our best model and then used the model to predict sales for the dates in the validation set.

NOTE: The data was split differently (using a different date) for this SARIMA analysis than how it was split for Random Forest Regression. Therefore the metrics below cannot be directly compared to the Random Forest Regression metrics.

Fitting the model and generating predictions produced the following metrics. The RMSE of 1615 is 15.5% of the average actual sales of $10,401 for Store 1 Department 3's. Future effort can attempt to fine tune the model to decrease the RMSE.

  • Mean Squared Error (MSE) = 2608873
  • Root Mean Square Error (RMSE) = 1615
  • Mean Absolute Error (MAE) = 1306

We created a function to plot predicted versus actual weekly sales. We created one plot to compare the predictions against the validation set. Then we created a second plot to visualize the predictions with the entire range of available dates.

#Fit the chosen model on the training set
mod = sm.tsa.statespace.SARIMAX(train_set,
                                #trend='c',
                                order=(1,0,0),
                                seasonal_order=(1,0,0,52))
model_fit = mod.fit()
forecast = model_fit.forecast(steps=len(test_set))

#Function to plot predicted vs actual weekly sales

def plot_pred_vs_actual_sales(actualdates,
                              actualsales,
                              preddates,
                              predsales,
                              storenum,
                              deptnum,
                              x_date_spacing=1):
    plt.figure(figsize=(10, 5))
    plt.xlabel('Week')
    plt.ylabel('Weekly Sales ($)')
    plt.title('Actual and Predicted Weekly Sales for Store #' + str(storenum) +
              " Dept #" + str(deptnum))
    plt.xticks(np.arange(0, len(actualsales)+1, x_date_spacing))    
    plt.xticks(rotation=45)    
    plt.plot(actualdates,
             actualsales,
             label='Actual Weekly Sales')
    plt.plot(preddates,
             predsales, 
             label='Predicted Weekly Sales', color='magenta')
    plt.legend()
    ax = plt.gca()
    ax.yaxis.set_major_formatter(tick.FuncFormatter(lambda x, p: format(int(x), ',')));

Model Fitting and Evaluation (Another Department)¶

We repeated the same steps with a different department, Store 33 Department 80. We first graphed this department's weekly sales and decomposed the data to see trend and seasonal components. Sales do not appear to be seasonal, however there is a clear decreasing trend.

We found the best model with the lowest AIC, then we fit the model and used it to generate predictions. The metrics are listed below. The RMSE of 1245 is 17.5% of the average actual sales of $7,093 for Store 33 Department 80. Future effort can attempt to fine tune the model to decrease the RMSE.

  • Mean Squared Error (MSE) = 1549237
  • Root Mean Square Error (RMSE) = 1245
  • Mean Absolute Error (MAE) = 1159

We noted that the predictions are higher than the actual sales. Removing the down trend from the data and then fitting the model again may improve predictions.
